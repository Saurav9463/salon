## PROBLEM A — Admin data not loading (Services, Team, Messages 
showing empty / Dashboard showing all zeros)

This is a Supabase RLS (Row Level Security) issue. The admin pages 
are fetching data but RLS is blocking reads because the user session 
is not being passed correctly, OR the Supabase queries have a bug.

### Fix A1 — Add error logging to every admin data fetch

In every admin page that fetches data, update the Supabase calls 
to log errors so we can see what's failing:

  supabase.from("services").select("*")
    .then(({ data, error }) => {
      if (error) console.error("Services fetch error:", error);
      setServices(data ?? []);
    });

Do this for ALL tables: services, team, messages, bookings, 
and all 6 count queries in the dashboard stats.

### Fix A2 — Fix dashboard stats count queries

The current count queries using { count: "exact", head: true } 
may not be returning data correctly after the api-client swap.

Replace ALL 6 dashboard stat queries with this pattern that 
fetches actual rows and counts them client-side (more reliable):

  useEffect(() => {
    const fetchStats = async () => {
      const [
        { data: allBookings },
        { data: pendingBookings },
        { data: confirmedBookings },
        { data: unreadMessages },
        { data: allServices },
        { data: allTeam },
      ] = await Promise.all([
        supabase.from("bookings").select("id"),
        supabase.from("bookings").select("id").eq("status", "Pending"),
        supabase.from("bookings").select("id").eq("status", "Confirmed"),
        supabase.from("messages").select("id").eq("read", false),
        supabase.from("services").select("id").eq("active", true),
        supabase.from("team").select("id").eq("active", true),
      ]);
      setStats({
        totalBookings: allBookings?.length ?? 0,
        pending: pendingBookings?.length ?? 0,
        confirmed: confirmedBookings?.length ?? 0,
        unreadMessages: unreadMessages?.length ?? 0,
        totalServices: allServices?.length ?? 0,
        teamMembers: allTeam?.length ?? 0,
      });
    };
    fetchStats();
  }, []);

## PROBLEM B — Mobile sidebar not closing

### Fix B1 — Sidebar must close when ANY nav item is tapped

Find the admin sidebar component. On EVERY nav link inside 
the sidebar, ensure onClick calls the close function:

  onClick={() => { 
    navigate(path);  // or however routing works
    setSidebarOpen(false); 
  }}

If the sidebar uses a prop to control open state, pass 
onClose={() => setSidebarOpen(false)} to the sidebar component 
and call it on every link click.

### Fix B2 — Backdrop must close sidebar

The dark overlay behind the open sidebar must close it on tap:

  {sidebarOpen && (
    <div
      className="fixed inset-0 bg-black/60 z-30 md:hidden"
      onClick={() => setSidebarOpen(false)}
      style={{ touchAction: "none" }}
    />
  )}

Make sure this div is rendered BEFORE (below in z-order) the 
sidebar, so tapping it registers correctly on mobile.

### Fix B3 — Sidebar z-index hierarchy (mobile)

Ensure this z-index order:
  - Backdrop overlay: z-30
  - Sidebar panel: z-40  
  - Hamburger button: z-50

### Fix B4 — Admin layout structure on mobile

The main content area must take FULL width on mobile when 
sidebar is closed:

  <main className="w-full md:ml-60 min-h-screen p-4 md:p-8 
                   pt-16 md:pt-8 transition-all duration-300">

Do NOT apply any margin-left on mobile — the sidebar should 
float OVER the content as an overlay, not push it.
