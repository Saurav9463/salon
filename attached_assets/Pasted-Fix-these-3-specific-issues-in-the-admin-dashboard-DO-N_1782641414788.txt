## FIX 1 — Admin page horizontal overflow / scroll

The entire admin content area has a horizontal scrollbar and 
cards are being cut off on the right side.

Root cause: the 3-column grid for services/team is overflowing 
the content area because the sidebar + grid total width exceeds 
the viewport.

In the admin Services page and any other admin page with a 
3-column grid:

Change the grid from:
  grid-cols-3
To:
  grid-cols-1 md:grid-cols-2 xl:grid-cols-3

Also wrap the entire admin content/main area with:
  className="w-full min-w-0 overflow-x-hidden"

And on the admin layout root container, add:
  style={{ maxWidth: "100%", overflowX: "hidden" }}

Also check if any container has a fixed width or min-width 
that could cause overflow and remove it.

## FIX 3 — Bookings table showing "—" for Service and Stylist

The booking exists but Service and Stylist columns show "—".
This means the foreign key join is not resolving correctly.

In the admin Bookings page, the fetch query must use explicit 
foreign key hints because Supabase has multiple relationships:

Replace the current bookings fetch with:

  supabase
    .from("bookings")
    .select(`
      *,
      service:services ( name ),
      stylist:team ( name )
    `)
    .order("created_at", { ascending: false })
    .then(({ data, error }) => {
      if (error) console.error("Bookings error:", error);
      setBookings(data ?? []);
    });

Then in the table row, display:
  Service column:  {booking.service?.name ?? "—"}
  Stylist column:  {booking.stylist?.name ?? "—"}

If the old booking was made before services/team were in the DB,
its foreign keys will be null and "—" is correct for that row.
New bookings made now will show the correct names.
