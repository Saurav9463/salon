The site is deployed on Vercel but shows a black screen with error:
"TypeError: l.map is not a function"

Root cause: Home.tsx, and other pages use hooks like useListServices() 
and useListTeam() from @workspace/api-client-react which call an 
Express API server at /api/... — this server does not exist on Vercel. 
When the API call fails, the hooks return null/undefined instead of 
an array, and .map() crashes the whole page.

The fix: Replace ALL api-client-react hooks across every page with 
direct Supabase queries. The supabase client already exists at 
@/lib/supabase.ts and is correctly configured.

---

## FILES TO FIX

### 1. src/pages/Home.tsx
Replace:
  import { useListServices, useListTeam } from "@workspace/api-client-react";
  const { data: servicesData } = useListServices();
  const { data: teamData } = useListTeam();

With direct Supabase queries using useEffect + useState:
  import { useEffect, useState } from "react";
  import { supabase } from "@/lib/supabase";

  const [servicesData, setServicesData] = useState([]);
  const [teamData, setTeamData] = useState([]);

  useEffect(() => {
    supabase.from("services").select("*").eq("active", true)
      .then(({ data }) => { if (data) setServicesData(data); });
    supabase.from("team").select("*").eq("active", true)
      .then(({ data }) => { if (data) setTeamData(data); });
  }, []);

  // Keep existing fallback logic:
  const services = servicesData?.length ? servicesData : FALLBACK_SERVICES;
  const team = teamData?.length ? teamData : FALLBACK_TEAM;
